Collection - HashSet 源码解析

# 概述

HashSet 实现了 Set 接口， 底层是 HashMap，可以存放 null 值，但只能有一个 null。HashSet 不保证元素的存放顺序和取出顺序是一致的，但取出顺序是固定的。HashSet 不能有重复的对象。

第一次添加时，table 数组扩容到16，临界值（threshold）是16*加载因子（loadFactor）是0.75=12；

如果 table 数组使用到了临界值12，就会扩容到16*2=32，新的临界值就是32*0.75=24，以此类推。

  • 添加一个元素时，先得到 hash 值，会转成索引值

  • 找到存储数据表 table，看这个索引位置是否已经存放有元素

  • 如果没有，直接加入

  • 如果有，调用 equals 比较，如果相同，就放弃添加；如果不同，则添加到最后

  • 在 Java8 中，如果一条链表的元素个数到达8，并且 table 的大小大于等于64，就会转化成红黑树，否则仍然会采用数组扩容机制。

public class HashSetStructure {
    public static void main(String[] args) {
        Node[] table = new Node[16];

        Node jack = new Node("jack", null);
        table[2] = jack;

        Node tom = new Node("tom", null);
        jack.next = tom;

        tom.next = new Node("join", null);

        Node cat = new Node("cat", null);
        table[3] = cat;

        System.out.println(Arrays.toString(table));

    }
}

class Node {
    String item;
    Node next;

    public Node(String item, Node next) {
        this.item = item;
        this.next = next;
    }
}
