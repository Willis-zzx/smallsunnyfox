Docker部署可执行jar包

# 前言

SpringBoot项目部署方式分为两种：

  1. 将项目打包成war包，部署到服务器上的tomcat服务器中
  2. 将项目打包成可执行jar包，然后将jar包上传至服务器上，然后java -jar 执行

Docker的发展给SpringBoot项目部署带了一种新的部署方式——容器化部署

容器化部署是将可执行jar包放到Docker中运行，主要有两种方式。

# 方式一：直接构建jar包运行的镜像

  1. 将项目打包，上传至服务器指定目录

  2. 在该目录下构建Dockerfile文件

    vim Dockerfile
    1

  3. Dockerfile写入如下指令

    FROM java:8
MAINTAINER demo
ADD demo-0.0.1-SNAPSHOT.jar demo.jar
EXPOSE 8080
ENTRYPOINT ["java","-jar","demo.jar"]
    1
    2
    3
    4
    5

    其中指令的含义：

    • FROM：基础镜像，FROM java:8 指的是拉取一个jdk1.8的镜像

    • MAINTAINER：作者，MAINTAINER demo 作者是demo

    • ADD：将打包的可执行jar包复制到镜像中并重命名（用 COPY 也可），ADD demo-0.0.1-SNAPSHOT.jar demo.jar将demo-0.0.1-SNAPSHOT.jar 复制到镜像中并重命名为 demo.jar EXPOSE: 声明端口

    • ENTRYPOINT ： 容器启动之后执行的命令，java -jar demo.jar 即启动jar

  4. 创建好Dockerfile文件后，执行命令构建镜像

    docker build -t demo .
    1

    注意：demo后空格，再加上.，

    注意最后的 . 表示Dockerfile在当前文件目录下。demo表示构建的镜像，构建成功后可以使用docker images命令查看镜像。

  5. 镜像构建成功后就可以运行容器

    docker run -d --restart=always --name demo -p 8080:8080 demo
    1

    其中参数的含义：

    • -d ：后台运行容器，并返回容器ID
    • –restart=always ：容器在停止或服务器开机之后会自动重新启动
    • -p ：指定端口映射，第一个8080指的是宿主机上的端口，第二个8080指的是jar包内tomcat的端口
    • 最后的demo指定镜像

  6. 容器启动后可以使用docker ps命令查看启动的容器

  7. docker logs --tail 300 -f 容器id，可以查看容器服务的日志

  8. 如果想更新jar包，只需要使用docker cp demo-0.0.1-SNAPSHOT.jar 容器ID:/demo.jar，就可以将demo-0.0.1-SNAPSHOT.jar拷贝进容器并重命名，然后 docker restart 容器ID 重启容器。

# 方式二：基于JDK镜像运行容器

  1. 在服务器中拉取jdk1.8的镜像

    docker pull jdk8
    1

  2. 创建目录，并将jar包上传至该目录

    cd /server/
mkdir deploy/jar
    1
    2

  3. 运行容器

    docker run -d \
--restart=always \
-v /server/deploy/jar:/jar
-v /server/logs/demo:/mnt/logs/demo \
-p 8080:8080 \
--name demo \
jdk8 /usr/bin/java -jar \
-Duser.timezone=GMT+08 \
/jar/demo-1.0.jar
    1
    2
    3
    4
    5
    6
    7
    8
    9

    命令说明：

    • -Duser.timezone ：指定时区
    • jdk8 /usr/bin/java -jar ：执行jar包
    • /jar/demo-1.0.jar ：指定需要执行的jar包的位置。

  4. 以后发布，只需要把宿主机目录里的jar包替换掉，重启容器。
